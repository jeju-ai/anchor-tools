# 앵커 사업단 AI 업무 룰북

제주한라대학교 RISE(앵커) 사업단 구성원이 Claude(AI)를 활용할 때 공통으로 적용되는 규칙.  
이 파일을 프로젝트 루트에 `CLAUDE.md`로 복사하면 모든 Claude 작업에 자동 적용된다.

---

## 1. 언어 및 문체

- **기본 언어**: 한국어
- **코드·커맨드·파일명**: 영어
- **영문 대학명**: **Cheju Halla University** (Jeju Halla University 사용 금지)
- **문체**: 공문서는 개조식(명사형 종결), 안내문은 평서체 허용
- **존댓말**: 외부 발신 문서는 격식 존댓말, 내부 메모는 간결체

---

## 2. 개조식 작성 규칙

한국 공공기관 표준 개조식 형식을 준수한다.

```
○ 대항목 (사업·안건 단위)
  - 중항목 (세부 내용)
    · 소항목 (수치, 부연)
```

### 종결어미 변환표

| 원문 | 개조식 |
|------|--------|
| ~했습니다 / ~하였습니다 | ~함 |
| ~입니다 / ~이다 | ~임 |
| ~할 예정입니다 | ~할 예정 |
| ~되었습니다 | ~됨 |
| ~추진하고 있습니다 | ~추진 중 |
| ~검토 중입니다 | ~검토 중 |

### 수치 표기 원칙

- 금액: `○○억 원` / `○○백만 원` (단위 앞 띄어쓰기)
- 날짜: `2026. 4. 5.(일)` 형식
- 비율: `○○%` (% 앞 띄어쓰기 없음)
- 인원: `○○명`

## 4. 예산 작업 규칙

- 집행률 계산: `집행액 ÷ 배정액 × 100`
- 과목 코드 기준: `budget-codes.md` 참조
- 연말 집행률 목표: **90% 이상**
- 불용액 10% 초과 시: 사업단장 보고 + 교육부 통보 필요

---

## 5. 보안 및 개인정보 보호

### 입력 금지 항목

절대로 AI에 입력하지 않는다:

- 개인정보: 주민등록번호, 전화번호, 주소, 생년월일
- 급여·인건비 개인 상세 내역
- 미공개 예산 계획 또는 확정 전 예산안
- 개인 면담·평가·인사 관련 내용
- 법적 분쟁 중인 사건의 상세 내용
- 타 기관의 비공개 정보

### 출력물 관리

- AI 생성 결과물은 반드시 사람이 검토 후 발송·제출
- 교육부 제출 문서: 담당자 + 부단장 검토 필수
- 외부 발송 공문: 서명 권자 최종 확인 후 발송

## 6. 파일 명명 규칙

```
YYMMDD-type-description-version.ext
```

예시:
- `260405-report-rise-y2-plan-v1.md`
- `260315-minutes-operations-committee.md`
- `260320-budget-q1-execution.md`

### 금지 사항

- 한글 파일명 (macOS NFD 분해 이슈)
- 공백 포함 파일명 (하이픈으로 대체)
- 특수문자 (`!`, `@`, `#` 등)

## 7. AI 활용 범위

### 권장 활용

- 보고서·공문서 초안 작성 및 개선
- 회의록 정리 및 요약
- 예산 집행률 계산 및 현황 표 생성
- KPI 달성률 분석 및 시각화
- 한영·영한 번역 (공식 검토 전 초안)
- 아이디어 브레인스토밍 및 구조화

### 주의 활용 (사람 검토 필수)

- 교육부 제출 공식 문서 최종본
- 법적 효력이 있는 계약·협약서
- 예산 확정 및 이체 관련 업무
- 인사·채용 관련 판단

### 금지 활용

- 개인정보 처리
- 미공개 내부 정보 외부 서비스에 입력
- AI 결과물을 검토 없이 그대로 공문 발송
