# 앵커 사업단 AI 활용 정책

**버전**: 1.0  
**시행일**: 2026. 4. 5.  
**관리**: 이영준 부단장 (CIO, hello@jeju.ai)

---

## 1. 목적

본 정책은 제주한라대학교 RISE(앵커) 사업단 구성원이 AI 도구(Claude, ChatGPT 등)를 업무에 활용할 때 지켜야 할 기준을 정함으로써:

- 업무 효율성 향상을 위한 AI 활용을 적극 장려하고
- 개인정보·기밀 정보의 유출을 방지하며
- AI 생성 결과물의 품질과 책임 소재를 명확히 하는 것을 목적으로 한다.

---

## 2. 적용 범위

앵커 사업단 소속 모든 구성원 (교수, 연구원, 행정원, 인턴 포함).

---

## 3. 허용 활용 범위

### 3.1 적극 권장 활용

| 활용 유형 | 설명 |
|-----------|------|
| 문서 초안 작성 | 보고서, 공문서, 안내문 초안 생성 |
| 회의록 정리 | 녹취 또는 메모를 정형화된 회의록으로 변환 |
| 번역 | 한영·영한 번역 (공식 발송 전 검토 필수) |
| 데이터 분석 | 예산 집행률, KPI 달성률 계산 및 시각화 |
| 아이디어 발상 | 브레인스토밍, 구조화, 기획서 틀 잡기 |
| 교육 자료 개발 | 강의 계획서, 학습 자료 초안 |
| 코딩 지원 | 업무 자동화 스크립트, 데이터 처리 코드 |

### 3.2 주의 활용 (사람 검토 필수)

| 활용 유형 | 주의 사항 |
|-----------|-----------|
| 교육부 제출 문서 | AI 초안 → 담당자 + 부단장 검토 후 제출 |
| 공식 보도자료 | AI 초안 → 홍보 담당자 검토 및 사실 확인 |
| 법적 문서 초안 | AI 초안 → 법무 검토 필수 (계약, 협약서) |
| 예산 관련 판단 | AI 분석 참고 → 회계 담당자 최종 확인 |

---

## 4. 금지 입력 항목

다음 정보는 어떠한 AI 서비스(Claude, ChatGPT, Gemini 등)에도 입력하지 않는다.

### 4.1 개인정보

- 주민등록번호, 여권번호
- 개인 연락처 (전화번호, 이메일, 주소)
- 생년월일 + 이름 조합
- 금융 계좌 정보

### 4.2 인사·급여 정보

- 개인별 급여·인건비 상세 내역
- 채용 심사 내용 및 평가 결과
- 개인 면담·징계 관련 내용
- 근무 평가 점수 및 순위

### 4.3 미공개 기밀 정보

- 확정 전 예산 계획안
- 교육부 협상 중인 내용
- 미공개 사업 계획 및 전략
- 타 기관의 비공개 정보 (협약 내용 포함)

### 4.4 법적 민감 정보

- 법적 분쟁 진행 중인 사건 상세
- 개인 고충 및 민원 내용

## 5. AI 생성 결과물 관리 원칙

### 5.1 검토 의무

- AI 생성 결과물은 반드시 사람이 내용을 검토하고 수정한 후 사용
- "AI가 썼다"는 이유로 검토를 생략하지 않음
- 결과물의 사실 확인 및 수치 검증은 작성자 책임

### 5.2 출처 표시

- 보고서 등 공식 문서에 AI 활용 여부를 기재할 필요는 없음
- 단, 내부 공유 시 "AI 초안 + 검토 완료" 표시 권장
- 학술 논문·학회 발표: 해당 학회·저널의 AI 활용 정책 별도 확인 필수

### 5.3 저작권

- AI 생성 텍스트의 저작권은 현행법상 불분명 — 핵심 아이디어·표현은 인간이 기여하도록 편집
- 학술 출판 시 AI 도구 활용 공시 여부는 해당 학술지 정책 따름

## 6. 데이터 보안

### 6.1 서비스 선택

| 서비스 | 업무 데이터 입력 가능 여부 |
|--------|------------------------|
| Claude (Anthropic) | 가능 (일반 업무 데이터) |
| Claude Teams/Enterprise | 가능 (민감도 낮은 내부 데이터) |
| ChatGPT Plus | 가능 (일반 업무 데이터) |
| 무료 AI 서비스 (미확인) | 금지 (학습 데이터 활용 가능성) |

### 6.2 입력 데이터 처리

- AI 서비스 입력 데이터는 해당 서비스의 개인정보처리방침에 따라 처리됨
- 민감 데이터는 익명화·추상화 후 입력 (예: 이름 → "A 연구원", 금액 → 비율로 변환)

---

## 7. 책임 및 위반 시 조치

### 7.1 책임 소재

- AI 생성 결과물을 사용한 업무의 책임은 해당 구성원에게 있음
- AI의 오류·환각(hallucination)으로 발생한 문제도 사용자 책임

### 7.2 위반 시 조치

| 위반 유형 | 조치 |
|-----------|------|
| 개인정보 입력 (경미) | 시정 지도 + 재발 방지 교육 |
| 개인정보 입력 (반복·중대) | 사업단장 보고 + 개인정보보호위원회 신고 검토 |
| 미공개 기밀 유출 | 즉시 보고 + 법적 조치 검토 |
