# 문서 분석 프레임워크

## 요약 구조 (Summary Structure)

### 표준 요약 템플릿

모든 문서 요약은 아래 구조를 기본으로 한다:

```markdown
## [문서명] 요약

**문서 유형**: [계획서 / 보고서 / 회의록 / 공문 / 제안서]
**작성일**: [날짜]
**작성 기관/부서**: [기관명]

### 핵심 요약 (3줄)
1. [가장 중요한 내용]
2. [두 번째 중요한 내용]
3. [세 번째 중요한 내용]

### 주요 내용
○ [대항목 1]
  - [세부 내용]
○ [대항목 2]
  - [세부 내용]

### 핵심 수치
| 항목 | 수치 | 비고 |
|------|------|------|
| [항목] | [값] | [출처/조건] |

### 결론 및 시사점
○ [결론]
○ [행동 필요 사항]

### 불명확/추가 확인 필요
- [확인이 필요한 항목]
```

## 교차 분석 방법 (Cross-Reference Method)

### 단계별 절차

**Step 1 — 개별 문서 파악**
- 각 문서의 핵심 주제 1~3개 식별
- 주요 수치, 날짜, 의사결정 사항 추출
- 문서 유형(계획/실적/지침/협약) 분류

**Step 2 — 공통 요소 매핑**
```
[문서A 키워드] ∩ [문서B 키워드] → 공통 주제
[문서A 수치] vs [문서B 수치] → 일치/불일치 확인
[문서A 날짜] ~ [문서B 날짜] → 시간적 연관성
```

**Step 3 — 불일치 플래그**
- 같은 항목에 대해 수치나 내용이 다른 경우 명시
- 최신 문서 우선, 단 이전 문서 내용도 함께 제시
- 예: "계획서(2026.01)에는 15억, 수정본(2026.03)에는 18억으로 변경됨"

**Step 4 — 종합 인사이트 도출**
- 전체 문서에서 반복되는 핵심 메시지
- 문서들이 암시하는 우선순위나 방향성
- 명시적으로 언급되지 않은 간극(Gap)

## 핵심 질문 템플릿 (Key Question Templates)

### 종합 분석용

```
"이 자료들의 공통된 핵심 주제 3가지는 무엇인가요?"
"전체 문서를 통해 가장 중요한 결정 사항 5개를 순위별로 정리해주세요."
"이 자료들이 제시하는 2차년도 방향성은 무엇인가요?"
"문서 간 상충되거나 불일치하는 내용이 있나요?"
```

### 예산·재무 분석용

```
"전체 예산 규모와 주요 항목별 배분은 어떻게 되나요?"
"집행률이 낮은 항목과 그 사유는 무엇인가요?"
"전년 대비 예산 변화가 큰 항목은 어디인가요?"
"추가 예산이 필요한 영역은 어디인가요?"
```

### 성과·KPI 분석용

```
"목표 대비 달성률이 가장 높은/낮은 지표는 무엇인가요?"
"미달성 지표의 공통 원인은 무엇인가요?"
"다음 분기에 집중해야 할 성과지표 3개는 무엇인가요?"
```

### 비교 분석용

```
"[문서A]와 [문서B]의 접근 방식 차이는 무엇인가요?"
"두 제안서 중 어느 것이 우리 기준에 더 부합하나요? 근거를 설명해주세요."
"1차년도와 2차년도 계획의 주요 변경 사항은 무엇인가요?"
```

### 보고서 생성용

```
"이 내용을 개조식 보고서로 변환해주세요."
"교육부 제출용 실적 보고 형식으로 재구성해주세요."
"핵심 내용을 임원 브리핑 1페이지로 요약해주세요."
"이 자료를 기반으로 FAQ 15개를 만들어주세요."
```

## NotebookLM vs Claude Projects 기능 비교

| 기능 | Google NotebookLM | Claude Desktop Projects |
|------|-------------------|------------------------|
| 문서 등록 수 | 최대 50개 소스 | 프로젝트당 다수 (용량 제한) |
| 지원 형식 | PDF, Google Docs, 웹, YouTube | PDF, 텍스트, 이미지, URL |
| 오디오 개요 | 지원 (팟캐스트 자동 생성) | 향후 지원 예정 |
| 한국어 품질 | 양호 | 우수 |
| 개조식 변환 | 미지원 | 지원 |
| HWP 지원 | 미지원 | 변환 후 지원 |
| 출처 인용 | 인라인 인용 지원 | 명시적 출처 표기 |
| 커스텀 지침 | 미지원 | System Prompt 지원 |
| 보고서 형식 | 미지원 | anchor-report 연동 |
| 프라이버시 | Google 서버 | Anthropic 서버 |
| 협업 | 노트북 공유 | 프로젝트 공유 제한적 |
| 비용 | 무료 (Google One) | Claude Pro/Team 구독 |

**선택 기준**:
- 민감 내부 문서 + 개조식 보고서 필요 → Claude Projects
- 공개 자료 + 오디오 요약 필요 → NotebookLM
- 빠른 단일 문서 요약 → 어느 쪽이든 가능

## 문서 등록 체크리스트

등록 전 확인 사항:

- [ ] 개인정보(주민번호, 연락처, 급여) 포함 여부 → 마스킹 후 등록
- [ ] HWP/HWPX → Markdown 변환 완료 (anchor-hwp 스킬 사용)
- [ ] Excel → CSV 변환 완료 (복잡한 수식 포함 시 값만 저장)
- [ ] 스캔 PDF → OCR 처리 완료 (텍스트 인식 불가 파일 제외)
- [ ] 문서 최신본 확인 (구버전 대신 최신 개정본 사용)
- [ ] 프로젝트 주제와 연관성 확인 (무관한 문서 제외)

## 분석 결과 활용

### 출력물 유형별 활용처

| 출력물 | 활용처 |
|--------|--------|
| 개조식 요약 | 회의 안건 자료, 내부 보고서 |
| 비교 분석표 | 의사결정 지원, RFP 평가 |
| FAQ | 신규 구성원 온보딩, 대외 안내 |
| 핵심 수치 정리 | 대시보드, KPI 보고 |
| 시사점 보고 | 전략 수립, 계획 수정 |

### 보관 및 공유

- 중요한 분석 결과는 MD 파일로 저장 → `admin/` 또는 `planning/` 폴더
- 팀 공유 시 Claude 대화 내보내기 또는 복사·붙여넣기
- vault 연동: 핵심 인사이트는 `/sync` 명령으로 vault에 추출
